Searching and Sorting

Write a Java (ver. 1.6) program to sort an interval of a given list of Comparable elements using binary insertion sort method. Binary insertion method is a modified insertion method that uses binary search to find the insertion point. Your implementation has to be stable (has to preserve the original order of equal elements). There are three parameters to your program: 
a - the list to be sorted 
left - interval starting position index (included) 
right - interval ending position index (excluded) 
You can only modify the given interval of the list.

public static <T extends Object & Comparable<? super T>> 
      void binaryInsertionSort (List<T> a, int left, int right)

Arrange a competition between different sorting methods:
  1. insertion sort
  2. quicksort
  3. binary insertion sort (your program)
  4. Java API built-in sort
Create a long random list of comparable elements and provide a copy of the same list to all methods during competition. Let the size of the interval to be sorted be 64000, 128000, 256000 and 512000 elements (if your computer is slow make these numbers smaller but arrange at least four rounds) . Measure the running time of each method for each competition round and put this data into a table in xls ("old" Excel) format. Create a line diagram to illustrate the growth of running time depending on amount of data. All four lines have to be on the same diagram (x-axis: amount of data, y-axis: running time in ms).

Measure only "clean" sorting time, average of several attempts gives even better result.

You have to upload both your program AND your xls-file with the diagram to Moodle in time!

Järjestamine ja otsimine


Koostada Java (ver. 1.6) meetod, mis järjestab etteantud võrraldavate (Comparable) elementide listis (java.util.List) etteantud vahemiku kahendpistemeetodil (s.t. pistemeetodil, mille korral pistekoht leitakse kahendotsimise abil). Realiseerida see meetod stabiilsena (s.t tuleb säilitada võrdsete elementide korral nende esialgne omavaheline järjestus). Meetodi parameetriteks on järjestatav list a, vahemiku algusindeks left (kaasa arvatud) ning lõpuindeks right (välja arvatud). 
Listi tohib muuta ainult etteantud vahemiku piires. 

public static <T extends Object & Comparable<? super T>> 
      void binaryInsertionSort (List<T> a, int left, int right)

Need meetodid on loengul käsitletud. Meetod tuleb realiseerida stabiilsena (säilitada võrdsete elementide korral nende esialgne omavaheline järjestus).

Korraldada "aus võistlus" meetodite vahel, mõõtes lahendamisaega ühe ja sama juhusliku listi  järjestamiseks erinevate meetoditega: pistemeetodil, kahendpistemeetodil (teie koostatud programm), kiirmeetodil ja Java APIs sisalduva meetodiga.

Andmemahud olgu näiteks  64000, 128000, 256000 ja 512000 elementi. Koostada lahendusaegadest tabel (soovitavalt "vanas" xls-formaadis) ja joongraafik, millel paigutada samasse teljestikku kõik neli joont (lahendamisaja sõltuvus andmemahust erinevate meetodite korral): x-teljel on andmemaht, y-teljel lahendusaeg.

Mõõta ainult järjestamiseks kuluvat aega iga meetodi jaoks võimalikult segamatult. Parema tulemuse saate, kui teete mitu mõõtmist ning leiate tulemustest keskmise.

Tähtaeg on teie rühma materjalide juures Moodle's, lahendused saata Moodle kaudu: programm automaattestina ja tabel/graafikud Excel failina.

Õigeaegse ja korrektse lahenduse eest kuni 6 punkti (3 - kahendpistemeetod, 3 - tabelid ja graafikud).
